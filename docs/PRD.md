Product Requirements Document (PRD)

Project Name: PulseKontrol (v2.0)

1. Цель продукта

Создать одностраничное веб-приложение (SPA) на React, агрегирующее топовые новости Hacker News и трендовые репозитории GitHub. Продукт фокусируется на премиальном визуальном опыте, предлагая две радикально отличающиеся атмосферы (Dark Cyber/Space и Light Glassmorphism) для технической аудитории, желающей мониторить тренды в эстетически приятном интерфейсе.

2. Целевая аудитория

Разработчики и инженеры (Software & AI/ML)

UI/UX дизайнеры и энтузиасты интерфейсов

Технические руководители (CTO, Team Leads)

Пользователи, ценящие "Clean Desk" эстетику

3. Основная идея

Пользователь получает мгновенный доступ к пульсу техно-индустрии. Интерфейс не просто отображает списки, а создает "иммерсивное пространство".

В темной теме: Ощущение командного центра космического корабля или IDE будущего.

В светлой теме: Ощущение чистоты и прозрачности в стиле Apple VisionOS (Glassmorphism).

4. Пользовательские сценарии (User Stories)

4.1 Мониторинг "Всего сразу" (Dashboard View)

Пользователь открывает вкладку "All Feeds" и видит две колонки (Hacker News и GitHub) одновременно для быстрого сканирования.

4.2 Фокусировка на источнике (Focus View)

При выборе вкладки "Hacker News" или "GitHub Live", выбранная колонка растягивается на всю ширину контейнера, а неактивная скрывается. Это позволяет углубиться в контент одного источника.

4.3 Эмоциональное переключение темы

Пользователь переключает тему не просто для изменения цвета, а для смены настроения. Кнопка переключения имитирует "мерцание лампочки" (flicker effect), добавляя тактильное ощущение механического переключателя.

5. Основные функции и навигация

5.1 Хедер и Навигация

Левая часть (Вкладки):

All Feeds: Активная по умолчанию, показывает обе панели.

Hacker News: Показывает только панель HN (full-width).

Midjourney: Декоративный элемент. Текст зачеркнут, курсор not-allowed (показывает, что функционал недоступен/отключен).

GitHub Live: Показывает только панель GitHub (full-width).

Правая часть (Статус):

Индикатор "Last Refresh" (время с последнего обновления).

Кнопка принудительного обновления (Refresh).

Переключатель темы с индикатором состояния (Желтый для Dark, Синий для Light).

5.2 Блок Hacker News (Realtime)

Отображает топ-15 горячих историй.

Элементы списка содержат:

Порядковый номер (Rank) в круге.

Заголовок новости.

Домен источника (автоматически извлекается из URL) в качестве подзаголовка.

Метаданные: Очки (Score), Комментарии, Время, Автор.

Первый элемент в списке может иметь особое визуальное выделение (акцент).

5.3 Блок GitHub (Trending)

Отображает репозитории, созданные за последние 7 дней, отсортированные по звездам (имитация Trending).

Элементы списка содержат:

Порядковый номер.

Полное имя репозитория (user/repo).

Язык программирования.

Описание (Description).

Метаданные: Звезды, время создания.

6. Визуальный стиль и UI/UX

6.1 Темная тема (Space/Cyber - Default)

Фон: Глубокий черный/темно-синий (#08090F) с использованием многослойных градиентов (Nebula effect) синего и фиолетового цветов. Добавлен шум (noise) для текстуры.

Контейнеры: Полупрозрачные темные блоки с тонкими границами (border-white/5).

Акценты: Неоновый синий, свечение (glow effects) при наведении.

Типографика: Светло-серая, контрастная.

6.2 Светлая тема (Apple Glassmorphism)

Фон: Чистый светлый градиент (White/Blue/Purple tint).

Контейнеры: Эффект матового стекла (backdrop-blur-xl), высокая прозрачность (bg-white/70), мягкие тени.

Акценты: Ярко-синий (Apple Blue).

Типографика: Темно-серая, четкая.

6.3 Анимации

Theme Toggle: Эффект "сломанной лампочки" (Flicker) при нажатии.

Tab Switching: Плавное скрытие/растягивание колонок (transition-all duration-500).

Hover: Подсветка границ и фона элементов списка.

7. Технические требования

7.1 Стек технологий

Framework: React (функциональные компоненты, Hooks).

Styling: Tailwind CSS (активное использование модификаторов dark:, backdrop-blur, group-hover).

Icons: Lucide React.

Structure: Single File Component (для удобства портативности и демонстрации).

7.2 Получение данных (API)

Метод: fetch (REST API).

Hacker News: Использование официального Firebase API (topstories.json + item/{id}.json).

GitHub: Использование Search API (q=created:>date, sort=stars). Примечание: Используется Search API как замена неофициальному Trending API для стабильности.

Автообновление: Каждые 10 минут.

7.3 Адаптивность

Desktop: Полноценный двухколоночный вид или широкий одноколоночный (в зависимости от вкладки).

Mobile: Табы (All/HN/GH) превращаются в горизонтальный скролл-бар или кнопки. Контент адаптируется под узкий экран, скрывая второстепенные метаданные при необходимости.

8. Показатели успеха

Пользователь получает актуальные данные без перезагрузки страницы.

Визуальный переход между темами вызывает "WOW-эффект".

Интерфейс остается отзывчивым и читаемым в обоих цветовых режимах.
